# **Understanding `Model_link`**

## **RECOMMENDED: READ ALONGSIDE CODE**

`Model_link` is the **GUI entry point** of the DNDC DLL (`DNDCgo.dll`), exported as ordinal `@1` in `DNDCgo.def`.

The GUI (`DndcGraphics`) calls it once per simulation cell per year-group, passing everything the model needs to run a single site.

It is declared with `WINAPI` (`__stdcall`) so that the calling convention matches what the GUI host expects ‚Äî see `DNDCgo.h:36`.

***

## **Inputs**

### GUI Link Structure

*   `MODEL_LINK_INFO* pLinkInfo` ‚Äî pointer to a struct defined in `DNDCgo.h:26‚Äì32` that carries four fields:
    *   `CWnd* pWnd` ‚Äî pointer to the MFC window (used for drawing progress/results back to the GUI)
    *   `CDC* pDC` ‚Äî pointer to the device context (used by the GUI paint routines)
    *   `int DNDC_or_Other` ‚Äî flag the DLL sets to tell the GUI whether a DNDC model or another model ran
    *   `int StopRun` ‚Äî the DLL sets this to `1` if the user clicked Stop; the GUI checks it between calls to abort the batch

### Spatial / Scale

*   `int scale` ‚Äî spatial scale of the run: `0` = regional/gridded, `1` = site-level point run
*   `float ha` ‚Äî area of the cell/site in hectares; used to scale fluxes
*   `int ProvinceOrder` ‚Äî index of the province within the batch list
*   `int S_county` ‚Äî county number within the province
*   `int CountyIDX` ‚Äî county index used for database lookups
*   `int landuse` ‚Äî land use type code (maps to a crop system or land cover class)

### Time

*   `int S_SoilYear` ‚Äî the year used to initialize soil state (spin-up year)
*   `int S_ThisYear` ‚Äî the current calendar year being simulated
*   `int S_SimuYears` ‚Äî total number of years to simulate for this cell
*   `int StartYear` ‚Äî the calendar year the simulation starts from

### Databases / Paths

*   `char* DatabaseName` ‚Äî name of the climate/soil database to read inputs from
*   `char* r_Province` ‚Äî province or region name string; used to locate regional input files
*   `char* ScenarioName` ‚Äî name of the management scenario; used for output file naming
*   `char* DirRoot` ‚Äî root directory path override (can redirect where DNDC reads/writes files)
*   `char f_region[6][150]` ‚Äî array of 6 file path strings (each up to 150 chars) pointing to regional input files
*   `char* DroughtYield` ‚Äî path to a drought yield reference file; only used when `DroughtID == 1`
*   `char* BatchPass` ‚Äî string passed through to `DNDC100go` for batch output naming/tracking

### Simulation Flags

*   `int ifdaily` ‚Äî `1` = write daily output files; `0` = skip daily output
*   `int MonteCarlo` ‚Äî `1` = this is a Monte Carlo uncertainty run
*   `int ZXH` ‚Äî internal index passed down into the core model (exact role is model-internal)
*   `int Batch` ‚Äî `1` = running in batch mode (multiple sites); affects output aggregation
*   `int GlobeData` ‚Äî flag indicating whether global-scale gridded input data is being used
*   `int sample` ‚Äî Monte Carlo sample number; distinguishes replicates in uncertainty runs
*   `int RRRRR` ‚Äî an internal flag passed into the core model (role is model-internal)
*   `int IrriType` ‚Äî irrigation type code (e.g., flood, drip, rainfed); affects water balance
*   `char* country_ID` ‚Äî country identifier string; used for regional database selection
*   `int livestock` ‚Äî `0` = no livestock, `>=1` = livestock run (selects `DNDC101go` instead of `DNDC100go`)
*   `int Monitor` ‚Äî monitoring/debug output flag; when set, extra diagnostic output is written
*   `int DroughtID` ‚Äî `1` = drought simulation mode; changes `DroughtCycle` and `S_SimuYears`
*   `int FFN` ‚Äî farm/field number passed into the core simulation

### Drought Setup (`DNDCgo.cpp:100‚Äì105`)


#### Drought skip logic (`DNDCgo.cpp:177‚Äì188`)

*   In drought mode, different drought cycles skip certain years using `goto DroughtScape` (jumping to the label at the bottom of the year loop, effectively skipping that year's simulation)
*   `drc==1` ‚Äî baseline cycle, no skipping
*   `drc==2 or 3` ‚Äî skip year 1 (start mid-drought)
*   `drc>=4` ‚Äî skip year 1 always; also skip if irrigated (`IrriType==1`)
*   This pattern generates varied drought exposure scenarios across the 9 cycles

#### Per-year model allocation and run (`DNDCgo.cpp:190‚Äì215`)

```cpp
class_model *DNDC;
if(!(DNDC = new class_model()))
    AfxMessageBox("No enough memory to run modelClass()");
```

A **fresh `class_model` is allocated every year** ‚Äî this is intentional. Each year starts with a clean state; continuity between years is handled through files (intermediate soil state files written by the previous year and read back by the next). This avoids accumulated state bugs across years.

```cpp
DNDC->year = ThisYear;
```

Sets the model's internal year index so it knows which year's climate/management data to load.

```cpp
DNDC->DNDC100go(pLinkInfo, scale, S_SoilYear, ha, cropping_system, ProvinceOrder,
    S_county, CountyIDX, landuse, MonteCarlo, scale, ThisYear,
    r_Province, ZXH, Batch, sample, RRRRR, IrriType, country_ID, livestock,
    Monitor, drc, DroughtID, f_region, DroughtYield, FFN, FarmField, BatchPass);
```

*   `DNDC100go` is the main cropland simulation function inside `class_model` (defined in `Dndc_main.cpp`)
*   All of `Model_link`'s parameters are forwarded into it ‚Äî the function itself does not compute anything; it is purely a dispatcher and allocator
*   `drc` (current drought cycle) and `FarmField` (current farm field index) are added on top of the original inputs

```cpp
delete DNDC;
```

Immediately frees the model instance after each year completes. Because a new one is `new`'d at the top of the next iteration, memory never accumulates across years.

```cpp
DroughtScape:;
```

The `goto` target label ‚Äî a no-op statement. Years that should be skipped in drought mode jump here, bypassing the `new`/`DNDC100go`/`delete` block entirely for that year.

***

### Return (`DNDCgo.cpp:229`)

```cpp
return(1);
```

Always returns `1` to the GUI caller. The GUI does not use the return value for error checking ‚Äî the `StopRun` field in `pLinkInfo` is the actual control channel between DLL and GUI.
